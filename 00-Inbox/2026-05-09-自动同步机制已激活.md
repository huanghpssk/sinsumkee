## 三、本地侧配置（黄总）

### 必备条件
- [ ] Git 已安装（Mac 自带，确认：`git --version`）
- [ ] Obsidian Git 插件已安装并启用
- [ ] Obsidian Git 自动 pull 间隔设为 **5 分钟**
- [ ] Obsidian Git **Pull on startup** 已开启

### 首次拉取（Vault 根目录执行）
```bash
cd "你的 Obsidian Vault 路径"
git init
git branch -m main
git remote add origin https://github.com/huanghpssk/sinsumkee.git
git pull origin main --allow-unrelated-histories
```

### 日常自动同步
- 启动 Obsidian → 自动 pull 云端最新内容
- 每 5 分钟 → 自动检查并拉取更新
- 手动强制同步 → `Cmd/Ctrl + P` → 输入 **"Obsidian Git: Pull"**

## 七、故障排查

| 问题 | 排查步骤 |
|------|----------|
| Obsidian 没收到新文档 | 1. 检查 Obsidian Git 是否启用<br>2. 手动执行 **"Obsidian Git: Pull"**<br>3. 检查网络连接 |
| GitHub 推送失败 | 1. 检查 PAT Token 是否过期<br>2. 仓库是否为 Private（是正常）<br>3. 网络代理设置 |
| 本地笔记和云端冲突 | 1. 先备份本地修改<br>2. 手动 `git pull` 处理冲突<br>3. 优先保留较新内容 |
| 目录乱了 | 保持 `00-Inbox` 入口习惯，定期整理到对应项目目录 |
